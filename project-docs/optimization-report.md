## AEO対策（Answer Engine Optimization）

### AEOとは？

AEOは「Answer Engine Optimization（回答エンジン最適化）」の略で、GoogleやChatGPTなどのAIアシスタントが**直接答えを提示する**際に、あなたのサイトが情報源として選ばれやすくするための対策です。

#### 従来のSEOとの違い

- **SEO**: 検索結果の**リスト**に上位表示させる
- **AEO**: AIが**直接答えを読み上げる・表示する**ときに、あなたのサイトの情報を使ってもらう

**例）ユーザーがAIに質問**
「北参道でペット同伴OKのカフェを教えて」

→ AIが「Green Beans Coffeeは北参道駅徒歩3分のペットフレンドリーなカフェです。店名看板はありませんが...」と**あなたのサイトの情報をもとに答える**

---

### 実施したAEO対策

#### 1. **構造化データ（JSON-LD）の実装** ★最重要

**実施内容:**
- **LocalBusiness（CafeOrCoffeeShop）**: 店舗の住所、電話番号、営業時間、位置情報など
- **FAQPage**: 15問のよくある質問と答え
- **WebSite**: サイトの概要とアンバサダー情報

**期待される効果:**
- Googleマップや音声検索で正確に表示
- AIアシスタントが信頼できる情報源として引用
- リッチリザルト（検索結果での拡張表示）の獲得

#### 2. **FAQ（よくある質問）セクションの充実**

**実施内容:**
15個の具体的な質問と答えを用意し、サイト上でも構造化データでも提供。

- 「ペットを連れて行っても大丈夫ですか？」
- 「グルテンフリーのメニューはありますか？」
- 「予約はできますか？」など

**期待される効果:**
- ユーザーの疑問を先回りして解決
- AIが「このサイトは信頼できる情報源」と判断
- 音声検索での直接回答に採用される可能性

#### 3. **自然で具体的な文章表現**

**実施内容:**
AIが引用しやすい「人間らしい、でも情報は正確」な文章。

- 地域名を自然に含める（「北参道」「千駄ヶ谷」「ダガヤサンドウ」）
- 特徴的なキーワードを織り交ぜる（「ペットフレンドリー」「グルテンフリー」「自家焙煎」）
- 実体験に基づいた親しみやすい表現

**期待される効果:**
- AIが「信頼できる実体験情報」と判断
- 自然言語処理での文脈理解向上

## SEO対策（Search Engine Optimization）

### 実施したSEO対策

#### 1. **タイトルタグ最適化**

**実施内容:**
```html
<title>Green Beans Coffee | 北参道・千駄ヶ谷のペットフレンドリー自家焙煎コーヒー専門店</title>
```

- 店名を最初に配置
- 地域名（北参道・千駄ヶ谷）を含める
- 主要キーワード（ペットフレンドリー、自家焙煎、コーヒー専門店）を自然に配置
- 60文字以内に収める（検索結果での切り捨て防止）

**効果:**
- 検索結果でのクリック率向上
- 複合キーワードでの上位表示

#### 2. **メタディスクリプション最適化**

**実施内容:**
```html
<meta name="description" content="北参道駅徒歩3分。店名看板のない隠れ家コーヒー専門店。ペット同伴OK、グルテンフリーメニューあり。アンバサダーKikumiが見つけた魅力をご紹介。">
```

- 具体的な情報（徒歩3分、ペット同伴OK、グルテンフリー）
- 独自性（店名看板のない、アンバサダー）
- 160文字以内に収める

**効果:**
- 検索結果でのクリック率向上
- ユーザーの期待値設定

#### 3. **見出しタグの階層構造**

**実施内容:**
- H1: サイトタイトル（1箇所のみ）
- H2: 各セクション見出し（About、Menu、Experience等）
- H3: サブセクション見出し
- 論理的な階層構造を維持

**効果:**
- 検索エンジンのコンテンツ理解向上
- アクセシビリティ向上

#### 4. **セマンティックHTML**

**実施内容:**
- `<header>`: ヘッダー領域
- `<nav>`: ナビゲーション
- `<main>`: メインコンテンツ
- `<section>`: 各セクション
- `<article>`: 独立したコンテンツ（Visit Diary等）
- `<footer>`: フッター領域

**効果:**
- 検索エンジンのページ構造理解
- スクリーンリーダーでの読み上げ改善

#### 5. **画像の最適化**

**実施内容:**
- 全画像に`alt`属性を設定（説明的なテキスト）
- `loading="lazy"`で遅延読み込み
- `decoding="async"`で非同期デコード
- WebP等の軽量フォーマット推奨

**効果:**
- 画像検索での表示
- ページ速度向上
- アクセシビリティ向上

#### 6. **内部リンク構造**

**実施内容:**
- スムーズスクロールによるセクション間リンク
- ナビゲーションから全セクションへのアクセス
- 外部リンクに`rel="noopener noreferrer"`を設定

**効果:**
- ユーザーの回遊性向上
- ページ内の重要度分散

#### 7. **Canonical URL設定**

**実施内容:**
```html
<link rel="canonical" href="https://gbc-fan-site.pages.dev/">
```

**効果:**
- 重複コンテンツ問題の回避
- 正規URLの明示

#### 8. **OGP（Open Graph Protocol）設定**

**実施内容:**
```html
<meta property="og:title" content="Green Beans Coffee | 北参道・千駄ヶ谷のペットフレンドリー自家焙煎コーヒー専門店">
<meta property="og:description" content="北参道駅徒歩3分。店名看板のない隠れ家コーヒー専門店...">
<meta property="og:image" content="...">
<meta property="og:type" content="website">
```

**効果:**
- SNSでのシェア時の視覚的魅力向上
- ソーシャルトラフィックの増加

#### 9. **モバイルフレンドリー対応**

**実施内容:**
- レスポンシブデザイン（モバイルファースト）
- タッチ操作に最適化されたUI
- ビューポート設定

**効果:**
- Googleのモバイルファースト インデックスに最適化
- モバイル検索順位の向上

## 独自ドメイン取得後の追加効果

現在は一時的なURL（`gbc-fan-site.pages.dev`）を使用していますが、**独自ドメインを取得すると以下の効果がさらに高まります：**

### 推奨ドメイン例
- `greenbeanscoffee.jp`
- `gbc-coffee.jp`
- `greenbeanscoffee.com`

### 追加効果

1. **信頼性の向上**
   - 公式感のあるドメインはAIも検索エンジンも「公式情報」として扱いやすい
   - ユーザーからの信頼獲得

2. **ブランディング**
   - 覚えやすく、口コミされやすい
   - 名刺やSNSプロフィールに記載しやすい

3. **SEO効果**
   - ドメイン名にキーワードを含めることで検索エンジンが内容を理解しやすい
   - 長期的なドメインオーソリティの構築

4. **SNSシェア率向上**
   - URLが短く美しいと、シェアされやすい
   - 視覚的な信頼感

### 移行作業

独自ドメイン取得後は、HTML内の4箇所のURL変更のみで即座に効果が得られます：
1. OGP og:url
2. OGP og:image
3. Canonical URL
4. JSON-LD WebSite schema の url

（変更箇所は既にHTMLコメントで明示済み）
