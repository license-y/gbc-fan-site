G. AEO（AI検索・音声検索最適化）

項目内容

speakable スキーマh1・article p を音声読み上げ対象に指定（Google Assistant等対応）

内部リンク運用新記事作成時に3〜4本のアンカーテキストリンク＋2〜3本の逆リンクを追加（ルール化・運用中）
エリアキーワード北参道・千駄ヶ谷・ダガヤサンドウ・表参道・渋谷・代々木・新宿 等を各記事に自然に挿入

L. 文体・コンテンツ運用ルール

項目ルール
執筆視点アンバサダー視点（外部の立場）。「〜と教えていただきました」「〜とのことです」
一人称「私」固定
読点（、）最小限に抑える（Kikumi文体の最大の特徴）
お客さん表記「お客さん」または「お客さま」を文脈で使い分け。「客」単体は使用禁止
記事文字数2,000文字程度（Q&A・まとめ含む）
カテゴリ数原則1つ・最大2つまで（1記事あたり）
内部リンク新記事 → 既存記事：3〜4本 / 既存記事 → 新記事（逆リンク）：2〜3本
画像Unsplash画像はローカル保存（public/assets/images/articles/）・200KB以内推奨
Q&AAEO対策のため各記事に4問程度を推奨
