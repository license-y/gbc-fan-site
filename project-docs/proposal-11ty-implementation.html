02　現状のサイトのままでは難しい理由

現在のメインサイトは「1ページ完結型」として、店舗情報・FAQ・構造化データが一箇所に集約された設計になっています。これはAIが情報を参照しやすいAEO対策として非常に効果的であり、検索エンジンからの評価も得やすい構成です。今回の「記事を増やしてSEOを強化する」という構想は、この土台をさらに活かす、方向性として最適な次の一手です。

ただし、現在の形式のまま記事を10本・20本と追加していくには、記事ごとにHTMLファイルを1枚ずつ手作業で作る必要があり、現実的に管理が困難になります。

記事を自由に増やせる仕組みとして一般的によく使われるのが WordPress です。ただしWordPressは、有料サーバーの契約・セキュリティ対策・定期的なアップデートなど、維持管理のコストと手間が継続的にかかります。また、現在のサイトのデザインや構成をWordPressに移行する作業自体も、かなりの労力を要します。

💡 解決策：「11ty（Eleventy）」という仕組みを導入する
記事をシンプルなテキスト形式（Markdown）で書くだけで、HTMLページが自動生成されます。サーバー不要・メンテ不要・無料で運用でき、現在のサイトと同じCloudflare Pagesで動作します。WordPressの複雑さを抱えずに、同じ目標を実現できる現実的な選択肢です。

⚠️ 技術的な注意点
現在のメインサイトと記事エリアを同じドメインで共存させる設計は、一見シンプルに見えますが、技術的に最も慎重さが求められる部分です。ちょうど、既存の建物に増築する際に基礎から確認が必要なように、既存サイトへの影響を確認しながら丁寧に設定を進める必要があります。この工程を確実に行うことが、後々のトラブルを防ぐ鍵になります。
