# Google Search Console 登録手順

**作成日**: 2026年1月8日
**対象サイト**: Green Beans Coffee ファンサイト (https://greenbeanscoffeeambassador.com)

---

## Google Search Console とは？

Google Search Consoleは、Googleが無料で提供するウェブマスター向けツールです。

### できること
- 検索パフォーマンスの追跡（どのキーワードで検索されているか、クリック率、掲載順位）
- インデックス状況の確認（Googleに正しく登録されているか）
- 構造化データのエラー検出（JSON-LDが正しく認識されているか）
- サイトマップの送信（クロールを促進）
- モバイルユーザビリティのチェック
- セキュリティ問題の通知

### なぜ必要か
- **SEO/AEO対策の効果測定に不可欠**
- データ蓄積に時間がかかるため、できるだけ早く登録すべき
- 問題が発生したときにGoogleから通知が届く

## 登録手順

### ステップ1: Googleアカウントの準備

1. Googleアカウントを持っていない場合は作成してください
   - https://accounts.google.com/

2. サイト管理用のGoogleアカウントでログインしてください

---

### ステップ2: Google Search Console にアクセス

1. 以下のURLにアクセスしてください
   - https://search.google.com/search-console/

2. 「今すぐ開始」をクリック

---

### ステップ3: プロパティを追加

1. 左上の「プロパティを検索」をクリック

2. 「プロパティを追加」を選択

3. プロパティタイプを選択:
   - **「URLプレフィックス」を選択**（推奨）
   - `https://greenbeanscoffeeambassador.com` と入力
   - 「続行」をクリック

**注意**: 「ドメイン」ではなく「URLプレフィックス」を選択してください。こちらの方が設定が簡単です。

---

### ステップ4: 所有権の確認

サイトの所有権を証明する必要があります。以下のいずれかの方法を選択してください。

#### 【推奨】方法1: HTMLタグで確認（最も簡単）

1. 「HTMLタグ」タブを選択

2. 表示されるメタタグをコピー
   ```html
   <meta name="google-site-verification" content="XXXXXXXXXXXXXXXX" />
   ```

3. サイトの `index.html` の `<head>` セクション内に貼り付け
   - 場所: `/public/index.html` の `<head>` 内（例: 5行目付近）

4. ファイルを保存してデプロイ（Cloudflare Pagesにプッシュ）

5. Google Search Consoleに戻り、「確認」ボタンをクリック

6. 「所有権を証明しました」と表示されたら成功

**確認後も、メタタグは削除しないでください**（削除すると所有権が失われます）

---

#### 方法2: DNSレコードで確認（Cloudflareを使用）

1. 「ドメイン名プロバイダ」タブを選択

2. TXTレコードが表示されます
   ```
   google-site-verification=XXXXXXXXXXXXXXXX
   ```

3. Cloudflareのダッシュボードにログイン
   - https://dash.cloudflare.com/

4. ドメイン `greenbeanscoffeeambassador.com` を選択

5. 「DNS」セクションに移動

6. 「レコードを追加」をクリック

7. 以下を入力:
   - タイプ: `TXT`
   - 名前: `@`
   - コンテンツ: `google-site-verification=XXXXXXXXXXXXXXXX`
   - TTL: Auto

8. 「保存」をクリック

9. Google Search Consoleに戻り、「確認」ボタンをクリック

10. DNS反映に数分〜数時間かかる場合があります

---

### ステップ5: サイトマップの送信

所有権の確認が完了したら、サイトマップを送信します。

1. 左側のメニューから「サイトマップ」を選択

2. 「新しいサイトマップの追加」欄に以下を入力
   ```
   sitemap.xml
   ```

3. 「送信」をクリック

4. ステータスが「成功しました」と表示されることを確認

**注意**: サイトマップが処理されるまで数時間〜数日かかる場合があります。

---

### ステップ6: 構造化データの確認

1. 左側のメニューから「拡張」→「構造化データ」を選択
   - または「エクスペリエンス」→「構造化データ」

2. 数日後、以下の構造化データが認識されているか確認:
   - `CafeOrCoffeeShop`（店舗情報）
   - `FAQPage`（よくある質問）
   - `WebSite`（サイト情報）
   - `Person`（アンバサダー情報）

3. エラーや警告がある場合は内容を確認して修正

---

### ステップ7: データの確認（数日後〜）

データが蓄積されるまで数日かかります。以下の項目を定期的にチェックしてください。

#### 検索パフォーマンス
- 左側のメニューから「検索パフォーマンス」を選択
- 以下の指標を確認:
  - **クリック数**: サイトがクリックされた回数
  - **表示回数**: 検索結果に表示された回数
  - **平均CTR**: クリック率（%）
  - **平均掲載順位**: 検索結果での平均順位

#### インデックス カバレッジ
- 「インデックス」→「ページ」を選択
- 「有効」のページ数を確認
- エラーがある場合は内容を確認

#### モバイルユーザビリティ
- 「エクスペリエンス」→「モバイルユーザビリティ」を選択
- エラーがないことを確認

## トラブルシューティング

### 所有権の確認に失敗する

**原因1**: メタタグが正しく配置されていない
- `<head>` セクション内に配置されているか確認
- ファイルがデプロイされているか確認（Cloudflare Pagesにプッシュ済みか）

**原因2**: DNS設定が反映されていない
- DNS変更は反映に時間がかかる（最大48時間）
- 数時間後に再度試してください

**原因3**: HTTPSでアクセスできない
- サイトがHTTPSで正しくアクセスできるか確認
- Cloudflare Pagesは自動的にHTTPS対応しています

### サイトマップの送信に失敗する

**原因1**: sitemap.xmlにアクセスできない
- ブラウザで直接アクセスして確認
  - https://greenbeanscoffeeambassador.com/sitemap.xml
- 404エラーが出る場合、ファイルが正しくデプロイされていません

**原因2**: XMLの形式が正しくない
- XMLのシンタックスエラーがないか確認
- オンラインバリデーターで検証: https://www.xml-sitemaps.com/validate-xml-sitemap.html

### 構造化データが認識されない

**原因1**: JSON-LDのシンタックスエラー
- 「拡張」→「構造化データ」でエラー内容を確認
- Googleの構造化データテストツールで検証: https://search.google.com/test/rich-results

**原因2**: データがまだ処理されていない
- 構造化データの処理には数日かかる場合があります
- 1週間後に再度確認してください

## おすすめの活用方法

### 週次チェック（毎週）
- 検索パフォーマンスの確認（クリック数、表示回数、CTR、掲載順位）
- どのキーワードで検索されているか確認
- 新しいエラーや警告がないか確認

### 月次チェック（毎月）
- 掲載順位の推移をグラフで確認
- FAQがリッチリザルトとして表示されているか確認
- モバイルユーザビリティの問題がないか確認

### データの活用
- 検索されているキーワードに基づいてコンテンツを追加・改善
- クリック率が低いページのメタディスクリプションを改善
- 掲載順位が低いキーワードに対してコンテンツを強化

## Google Analytics との連携（オプション）

Google Search Console と Google Analytics を連携すると、より詳細な分析が可能になります。

### 連携手順

1. Google Analytics にログイン
   - https://analytics.google.com/

2. 左下の「管理」をクリック

3. 「プロパティ設定」→「Search Consoleのリンク」を選択

4. 「リンク」ボタンをクリック

5. Google Search Consoleのプロパティを選択

6. 「確認」→「送信」をクリック

### 連携後のメリット
- Google Analytics内でSearch Consoleのデータを確認できる
- 検索キーワードとサイト内行動を一緒に分析できる
- より詳細なSEO分析が可能

## まとめ

### ✅ 必須のステップ
1. Google Search Console にサイトを登録
2. 所有権を確認（HTMLタグ or DNS）
3. sitemap.xml を送信
4. 構造化データが認識されているか確認（数日後）

### ⏰ データ蓄積期間
- 初回データ表示: 1〜3日
- 十分なデータ蓄積: 1〜3週間
- 傾向分析が可能: 1〜3ヶ月

### 🎯 期待される効果
- 検索パフォーマンスの可視化
- インデックス状況の確認
- 構造化データの検証
- SEO/AEO対策の効果測定
- データに基づいた改善施策の実施
