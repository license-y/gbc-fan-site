thirdplacejapan.com 1本に絞る理由

比較項目 thirdplacetokyo.net thirdplacejapan.com ✓
カバーエリア 東京限定 日本全国
インバウンド需要 高い さらに高い
将来の拡張性 東京のみ 大阪・京都・全国へ展開可能
ドメイン拡張子 .net（2番手） .com（最高信頼性）
競合の少なさ ほぼゼロ 完全ゼロに近い
ブランドの格 ローカル ナショナル
東京メインのコンテンツでも「japan」ドメインは全く問題なし。
Time Out Tokyo・Tokyo Weekenderなど東京中心でも「Japan」を名乗るメディアは多く、ユーザーも違和感を持たない。インバウンド向けには「Japan」のほうが検索されやすく、東京コンテンツで十分対応できる。

Section 02

サイト構築の全体手順

全体の流れ

① GitHubリポジトリ作成
↓
② サイト構築（11ty + Tailwind CSS）
↓
③ Cloudflare Pages にデプロイ
↓
④ 独自ドメイン（thirdplacejapan.com）を接続
↓
⑤ 記事を書き始める
1
GitHubリポジトリ作成
  • GitHub で新しいリポジトリを作成（例：thirdplacejapan）
  • GBCサイトのリポジトリ構成を参考にフォルダを作る
  • README.md にサイトの概要を記載しておく
2
サイト構築（GBCサイトと同じ技術スタック）
技術 用途 GBCサイトと同じ
11ty（Eleventy） 記事管理・静的サイト生成 ✓ 同じ
Tailwind CSS v4 デザイン ✓ 同じ（カラーのみ変更）
Pagefind 記事検索 ✓ 同じ
Animate.css / AOS スクロールアニメーション ✓ 同じ
JSON-LD構造化データ AEO対策 ✓ 同じ（内容を変更）

ビルドコマンド：npm run build（本番） / npm run dev（ローカル確認）

3
Cloudflare Pages にデプロイ
  • Cloudflare Pages の管理画面でGitHubリポジトリを連携
  • ビルド設定：コマンド npm run build・出力フォルダ public
  • GitHubにプッシュするたびに自動デプロイされる
4
独自ドメイン接続（thirdplacejapan.com）
  • Cloudflare Pages の管理画面 → 「カスタムドメイン」を追加
  • thirdplacejapan.com を入力して保存
  • DNSレコードを設定（Cloudflareでドメイン管理している場合は自動設定）
  • SSL証明書も自動発行される（httpsが自動で有効になる）
5
記事を書き始める
  • GBCサイトと同じMarkdownファイル形式で記事を作成
  • Claude Code（AI）と一緒に記事を執筆
  • GitHubにプッシュ → Cloudflare Pagesが自動ビルド → 公開

  • Accent：#4a8abf（ライトブルー）
  • Background：#f4f7fa（ブルーオフホワイト）

CLAUDE.md の設定（重要）

GBCサイトに CLAUDE.md があるように、thirdplacejapan.com 用にも専用の CLAUDE.md を作成する。Claude Code との作業品質を一定に保つために必須。

  • 東京エリア特集記事を20本公開
  • GBCサイトとの相互リンクを本格始動
  • 英語記事5本（Tokyo third place・pet-friendly cafe優先）
  • FAQ・JSON-LD構造化データを全記事に設置
3
フェーズ3：権威確立（3〜12ヶ月）
  • 記事数50本・FAQ累計200問を目標
  • 英語記事15〜30本（Tokyo third place 英語AEO引用85%を目指す）
  • 「日本のサードプレイスといえばthirdplacejapan.com」の認知形成
  • 情報コントロール度40〜70%を達成

まとめ

thirdplacejapan.com はGBCファンサイトと同じ技術スタック（11ty + Cloudflare Pages）で構築できるため、すでにノウハウがある状態から始められます。

コンテンツは東京メインで始め、将来的に全国へ拡張できる設計にすることで「日本のサードプレイス専門メディア」としての権威を段階的に確立していきます。

「サイトを作りたい」と言うだけでClaude Codeが構築を始められます。いつでもスタートできます。
