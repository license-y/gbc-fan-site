01　サイトの仕組み（全体像）

GBCコラムは「11ty（Eleventy）」という仕組みで動いています。記事をテキストファイル（Markdown）で書くと、自動的にウェブページが生成されます。

記事を公開するまでの流れ
1
Markdownファイルを作成・編集するsrc/articles/ フォルダに .md ファイルを追加するだけ
2
GitHub にアップロードする（push）Claude Code から「コミット＆プッシュ」を指示する
3
Cloudflare Pages が自動でビルド＆公開する数分後に greenbeanscoffeeambassador.com/articles/ に反映される

💡 Cloudflare Pages のビルドとは？
GitHubにアップすると、Cloudflare Pagesが自動的に npm run build を実行します。このコマンドが「MarkdownファイルをHTMLに変換する」「検索インデックスを生成する」「記事サイトマップを更新する」をすべて自動で行います。

02　記事を追加する方法
ファイルの場所

記事ファイルはすべて src/articles/ フォルダに置きます。

src/
└── articles/
    ├── pet-cafe-sendagaya.md      ← サンプル記事①
    ├── glutenfree-focaccia.md     ← サンプル記事②
    ├── roasting-experience.md     ← サンプル記事③
    └── 新しい記事.md              ← ここに追加していく
ファイル名のルール

ファイル名がそのままURLになります。英数字とハイフン（-）のみ使用してください。日本語・スペースは使用不可。

ファイル名URL
pet-cafe-sangubashi.md/articles/pet-cafe-sangubashi/
coffee-how-to-brew.md/articles/coffee-how-to-brew/
記事ファイルの書き方

ファイルの先頭に「---」で囲まれた設定部分（フロントマター）を書き、その後に本文を書きます。

---
layout: article.njk
title: ここに記事のタイトル
description: ここに記事の要約（検索結果やSNSシェア時に表示されます）
date: 2026-05-07
tags:
  - articles
  - ペットとコーヒー
thumbnail: /assets/images/hero_bg-1.jpg
---

ここから本文を書きます。

## 見出し（大）

本文の段落です。

### 見出し（中）

**太字** は重要な部分に使います。
フロントマターの各項目
項目必須説明
layout必須必ず article.njk と書く（変更不要）
title必須記事のタイトル
description推奨100〜120文字の要約。SEO・SNSシェアに使用
date必須公開日（YYYY-MM-DD形式）
tags必須articles は必ず含める。カテゴリを追加（後述）
thumbnail任意サムネイル画像のパス。なくてもOK
03　カテゴリ（タグ）の管理

現在使用中のカテゴリ：

ペットとコーヒー グルテンフリー 体験・イベント

新しいカテゴリを追加したい場合は、tags: に新しい名前を書くだけで自動的に作成されます。

tags:
  - articles
  - コーヒー      ← 新カテゴリを追加するだけでOK

📌 articles タグは必ず入れてください。これがないと記事一覧に表示されません。

06　記事を公開する手順（毎回の作業）
1
記事ファイルを作成・編集するsrc/articles/ フォルダに .md ファイルを追加または編集
2
ローカルで確認する（任意）npm run dev でブラウザ確認。問題なければ次へ
3
Claude Code に「コミット＆プッシュ」を指示する「記事を追加したのでコミット＆プッシュしてください」と伝えるだけ
4
Cloudflare Pages の自動デプロイを待つ通常3〜5分で公開される。ダッシュボードの Deployments タブで確認可

📌 新しい記事を追加するたびに、検索インデックスと記事サイトマップも自動で更新されます。手動での追加作業は不要です。

07　Google Search Console へのサイトマップ登録

記事を検索エンジンに早く認識してもらうために、2つのサイトマップを Google Search Console に登録してください。

サイトマップURL内容更新
/sitemap.xmlメインサイト・/articles/ 一覧手動（大きな変更時のみ）
/articles-sitemap.xml個別記事のURL一覧自動（記事追加時に自動更新）

登録手順： Search Console → サイドメニュー「サイトマップ」→ URLを入力して送信

08　記事内 FAQ について（AEO対策）

各記事の末尾に「よくある質問」セクションを入れることを推奨します。AIや検索エンジンが回答として引用しやすくなります。

## よくある質問

**Q. 質問をここに書きます**
はい、回答をここに書きます。具体的な情報（数字・固有名詞）を含めると効果的です。

**Q. 別の質問**
回答。

💡 メインサイトのFAQと内容が重複しても問題ありません。記事の文脈に特化した質問を入れることで、ロングテール検索への対応になります。

09　今後の運用について
Articles ナビリンクの変更タイミング

現在、メインサイトのナビの「Articles」はページ内の GBCコラムセクション（#articles）にスクロールする設定になっています。記事が10本を超えたら、直接 /articles/（記事一覧ページ）へ飛ぶよう変更することを推奨します。

変更箇所：public/index.html のナビ内 href="#articles" を href="/articles/" に2箇所変更するだけです。

記事URLとカテゴリの命名について

現在、ナビや一部の表示でカタカナ・英語が混在しています（例：「Articles」「ペットとコーヒー」）。全体的な表記の統一はオーナーと相談して決定してください。変更の際はカテゴリ名（タグ）の変更が必要になります。
