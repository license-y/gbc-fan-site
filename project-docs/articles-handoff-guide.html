00　作業開始前の必須確認

⚠️ 複数人で管理しているため、作業を始める前に必ず最新のファイルを取得してください。取得せずに作業を始めると、他の人の変更が上書きされてしまう場合があります。

作業開始時に必ず行うこと

Claude Codeのチャット欄に以下を入力して、Enterキーを押します。

あなた
! git pull origin main

💡 「!」（ビックリマーク）について：Claude Codeのチャット欄で ! に続けてコマンドを入力すると、ターミナルのコマンドとして直接実行されます。許可を求める画面が出たら「許可」をクリックしてください。

「Already up to date.」と表示されれば最新の状態です。そのまま作業を続けてください。

ファイルのダウンロードが始まった場合は、完了を待ってから作業を開始してください。

01　サイトの仕組み（全体像）

GBCコラムの記事は、Claude Code（AI）に指示するだけで作成・公開できます。Markdownファイルを自分で書く必要はありません。Claudeが記事を書き、GitHubへのアップロードも代行してくれます。

記事を公開するまでの流れ（全体）
1
Claude Code に「記事を書きたい」と伝える書きたい内容のメモを伝えるだけでOK
2
Claudeの質問に答えるカテゴリ・まとめの有無・Q&Aの有無・画像について5項目
3
下書きを確認・修正する気になる箇所を伝えれば何度でも調整してもらえる
4
「コミット＆プッシュ」を指示するCloudflare Pagesが自動でビルド＆公開。数分で反映される
02　記事を作る流れ（具体例）
ステップ 1：「記事を書きたい」と伝える
あなた
記事を書きたいです。GBCはコーヒーフレッシュを使わずに本物の牛乳を使っています。ホットコーヒーには温めて提供してくれます。こういうお店は年に一度出会えるかどうかのレベルです。
ステップ 2：Claudeから5つの質問が来る
Claude
ありがとうございます。記事を作成する前に確認させてください。

1. カテゴリ：以下の7つからどれにしますか？
　コーヒー ／ 焙煎 ／ グルテンフリー ／ 体験・イベント ／ ペット ／ 美容健康 ／ ダガヤサンドウ

2. まとめセクションをつけますか？（つける／つけない）

3. 記事末尾にQ&Aをつけますか？（つける／つけない）

4. 画像：提供しますか？Unsplashで探しますか？
ステップ 3：答えを返す（メモ程度でOK）
あなた
カテゴリはコーヒー。まとめはなし。Q&Aはつけてください。画像はUnsplashで探してください。
ステップ 4：下書きが届く → 確認・修正

Claudeが記事の下書きを作成します。気になる部分があれば、そのまま伝えるだけで修正してもらえます。

あなた
冒頭の書き方を変えてほしいです。実際には「コーヒーフレッシュはありません」とは言われていないので。

💡 文体のルール：記事はすべて「GBCアンバサダーとして体験・聞いた」という立場で書きます。店側の公式説明のような書き方（「〜を提供しています」「〜にこだわっています」）にならないよう、Claudeが自動的に確認・修正します。修正した場合はどこをどう直したか報告します。

ステップ 4-B：ローカルで記事の見た目を確認する（任意）

コミット＆プッシュの前に手元でレイアウトを確認したい場合は、Claudeに依頼できます。

あなた
ローカルで確認したいので開発サーバーを起動してください。

Claudeが npm run dev を実行し、確認用のURLを教えてくれます。ブラウザでそのURLを開いて記事の見た目を確認してください。確認が終わったら次のように伝えます。

あなた
確認できました。サーバーを止めてください。

⚠️ ローカル確認時の制限：開発サーバー（npm run dev）では記事のレイアウト・文章は確認できますが、検索機能は動作しません。検索もあわせて確認したい場合は「フルビルドでも確認したい」とClaudeに伝えてください。

💡 確認用URLの例：http://localhost:8080/articles/（記事一覧）または http://localhost:8080/articles/記事のスラッグ名/（個別記事）。正確なURLはClaudeが教えてくれます。

ステップ 5：公開する
あなた
OKです。コミット＆プッシュしてください。

以上で完了です。数分後にサイトに反映されます。

03　カテゴリ（7つ固定）

カテゴリは以下の7つに固定されています。勝手に増やさず、必ずこの中から選んでください。

コーヒー 焙煎 グルテンフリー 体験・イベント ペット 美容健康 ダガヤサンドウ

📌 カテゴリを追加・変更したい場合は「○○というカテゴリを追加したい」とClaudeに伝えてください。CLAUDE.mdも含めて一括で更新します。

04　画像の追加方法
自分で撮影した写真を使いたい場合
1
public/assets/images/articles/ フォルダに画像を置くファイル名は仮のままでOK（Claudeがリネームします）
2
Claudeに「画像を置きました」と伝えるファイルサイズを確認してもらえます
3
200KB超の場合は圧縮を案内されるTinyPNG（https://tinypng.com/）などで圧縮してから再度置く
4
Claudeが記事に合った名前にリネームして設定してくれる自分でファイル名を考える必要なし
フリー素材（Unsplash）を使いたい場合

「Unsplashで探してください」と伝えるだけで、Claudeが記事に合う写真を検索・ダウンロード・設定まで行います。

💡 画像の命名ルール：記事内容を表す英語名・ハイフン区切り・小文字（例：handdrip-event.jpg）でClaudeが統一して管理します。

05　記事を公開する（コミット＆プッシュ）

記事の内容がOKになったら、以下を伝えるだけです。

あなた
コミット＆プッシュしてください。

Cloudflare Pagesが自動でビルド＆公開します。通常3〜5分で反映されます。

📌 コミットだけ（公開前の保存）とコミット＆プッシュ（即公開）は別です。確認してから公開したい場合は「コミットだけしてください」と伝えてください。

📌 新しい記事を追加するたびに、検索インデックスと記事サイトマップも自動で更新されます。手動での追加作業は不要です。

技術補足

「コミット＆プッシュ」とは、ファイルの変更をGitHubに保存・送信することです。Cloudflare PagesはGitHubへの変更を検知して自動的に npm run build（MarkdownをHTMLに変換・検索インデックス生成・サイトマップ更新）を実行します。

06　Google Search Console へのサイトマップ登録

記事を検索エンジンに早く認識してもらうために、2つのサイトマップを Google Search Console に登録してください。

サイトマップURL内容更新
/sitemap.xmlメインサイト・/articles/ 一覧手動（大きな変更時のみ）
/articles-sitemap.xml個別記事のURL一覧自動（記事追加時に自動更新）

登録手順： Search Console → サイドメニュー「サイトマップ」→ URLを入力して送信

07　記事内 FAQ について（AEO対策）

各記事の末尾に「よくある質問」セクションを入れることを推奨します。AIや検索エンジンが回答として引用しやすくなります。記事作成時にClaudeへ「Q&Aをつけてください」と伝えれば自動で作成してくれます。

技術補足：Markdownでの記述例
## よくある質問

**Q. 質問をここに書きます**
回答をここに書きます。具体的な情報（数字・固有名詞）を含めると効果的です。

**Q. 別の質問**
回答。

💡 メインサイトのFAQと内容が重複しても問題ありません。記事の文脈に特化した質問を入れることで、ロングテール検索への対応になります。

08　今後の運用について
Articles ナビリンクの変更タイミング

現在、メインサイトのナビの「Articles」はページ内の GBCコラムセクション（#articles）にスクロールする設定になっています。記事が10本を超えたら、直接 /articles/（記事一覧ページ）へ飛ぶよう変更することを推奨します。

変更の際はClaudeに「Articlesのナビリンクを記事一覧ページに変更してください」と伝えるだけです。

技術補足：変更箇所

public/index.html のナビ内 href="#articles" を href="/articles/" に2箇所（デスクトップ・モバイル）変更します。
