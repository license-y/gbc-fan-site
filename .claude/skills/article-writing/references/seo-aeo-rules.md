# SEO・AEO記事作成指針

**店名の表記ルール（SEO対策・必須）：**

記事内での店名は以下の3種類を使い分ける。検索エンジンとAIに「同じ店の複数の呼び方」として認識させることがSEO・AEO対策に有効。

| 表記 | 使い方 |
|-----|--------|
| **Green Beans Coffee** | メインの表記。記事冒頭・見出し・まとめ・Q&Aなど、重要な箇所で使う |
| **グリーンビーンズコーヒー** | 日本語検索ヒット用。本文中に自然な形で数回使う（1記事あたり3〜5回を目安） |
| **GBC** | 一般の読者は知らない略称のため使用を控える。他記事の実際のタイトルをリンクアンカーテキストとして引用する箇所（既存タイトルに`GBC`が含まれる場合）のみそのまま使ってよい |

**適用ルール（2026-07改訂）：**
- 1記事の中で `Green Beans Coffee` と `グリーンビーンズコーヒー` の2種類を自然な配分で使う（タイトル・見出し・答えリード・まとめ・FAQも対象）
- 段落の冒頭・文の主語になる箇所、タイトルや見出しは `Green Beans Coffee` または `グリーンビーンズコーヒー` を優先
- **`GBC` は本文・タイトル・見出し・FAQ・画像altでは使わない**：一般の読者は略称を知らないため、初見で意味が伝わらない。`thumbnailAlt`や本文中のインライン画像`![alt](...)`のalt文も対象。既存記事へのリンクでその記事の実際のタイトルに`GBC`が含まれる場合は、リンクアンカーテキストとしてのみそのまま引用してよい（タイトルを改変すると誤ったリンク表示になるため）
- 不自然な詰め込みはしない。「グリーンビーンズコーヒーのグリーンビーンズコーヒーが…」のような繰り返しは避ける
- **背景**：2026-07に新規記事8本で`GBC`を多用していたところ「一般の読者は略称を知らない」との指摘を受け、本文からGBCを排除しGreen Beans Coffee／グリーンビーンズコーヒーの2表記に絞る方針へ変更した
- **既存記事の一斉監査（2026-07）**：方針変更前に書かれた記事が未対応のまま残っていたため、記事サイト全体でGBCの裸表記が本文63記事・description6記事・タイトル8記事に残存していることが判明し一括修正した。タイトルは他記事からのリンクアンカーテキストとして引用されているため現状維持とし、本文とdescriptionのみ修正する方針とした（影響範囲を考慮した判断）。今後も既存記事を編集する際にGBCの裸表記を見つけたら本文・description・alt文は修正してよいが、タイトル変更は他記事への影響を確認してから判断すること

**タイトル（SEO最重要項目）：**

記事タイトルはSEO対策の中で最も重要な要素。以下の3つのキーワードグループを**必ず自然な形でタイトルに含める**こと。

**① カテゴリワード（記事のカテゴリに対応するワードを1つ）：**
- コーヒー・カフェ / 焙煎 / グルテンフリー / 体験・イベント / ペット / 美容健康 / ビジネス / サードプレイス / ダガヤサンドウ
- 例：「北参道のコーヒー専門店で〜」「千駄ヶ谷のペット同伴カフェで〜」

**② 近隣地名（1〜2つ）：**
- 北参道 / 千駄ヶ谷 / 原宿 / 代々木 / 渋谷 / 表参道 / 青山
- タイトルに地名が入ることで「エリア検索」にヒットしやすくなる

**③ 地域施設・周辺スポット（自然に入れられる場合）：**
- 国立競技場 / 東京体育館 / 明治神宮 / 明治神宮外苑 / 新宿御苑 / 鳩森八幡神社 / 代々木公園
- 施設訪問後の立ち寄りスポットとして検索されやすくなる
- 無理に入れず、記事の内容と自然に結びつく場合のみ使用する

**タイトル作成のルール：**
- 「〜してみた」「〜に行ってきました」「〜が最高だった」など体験型タイトルはクリックされやすく、AIにも引用されやすい
- 30〜35文字以内を目安（長すぎると検索結果で切れる）
- 検索意図を想定して立てる（「北参道 コーヒー おすすめ」「千駄ヶ谷 ペット カフェ」など）
- **「東京」または「渋谷」をタイトルに自然に入れる** → 文脈に合う場合は「東京・北参道の〜」「渋谷・北参道の〜」など冒頭に置くと効果的。ただし無理に入れると不自然になるため、すべてのタイトルに必須ではない。自然に溶け込む場合のみ使う
- 下書きを書き始める前に、SEO/AEOを意識したタイトル案を2〜3個提示してユーザーに選んでもらう

**FAQ（Q&A）の書き方：**
- AIや音声検索が「そのまま読み上げられる」簡潔な回答を心がける
- 質問は実際に検索されうる形で立てる。例：
  - 「東京・北参道でおすすめのコーヒー専門店は？」
  - 「千駄ヶ谷でスペシャルティコーヒーが飲めるカフェは？」
  - 「ダガヤサンドウエリアのペットと入れるカフェは？」
- 質問文に「東京」「渋谷」を自然に含めるとSEO効果が高まる。ただし全問に入れると不自然になるため、1記事あたり2〜3問を目安に、文脈に合う箇所だけに使う
- 回答は2〜3文に収める。専門性と信頼性を感じさせる語尾（「〜です」「〜できます」）で締める
- 回答の冒頭でそのまま答えを述べる（前置き不要）。例：「Green Beans Coffeeがおすすめです。北参道駅から徒歩3分、自家焙煎のスペシャルティコーヒーを楽しめます。」

**「渋谷」「東京」「新宿」ワードの必須挿入（SEO・AEO戦略・最重要）：**

GBCは渋谷区に所在する。「渋谷」「東京」「新宿」は検索ボリュームが大きく先行優位性が高いため、全記事に必ず自然な形で入れること。

- **「渋谷」** → 全記事に必ず1回以上入れる。「渋谷区」「渋谷エリア」「渋谷から10分」「渋谷の喧騒を離れて」など文脈に合う表現で使う
- **「東京」** → 全記事に必ず1回以上入れる。「東京カフェ」「東京のサードプレイス」「東京・渋谷区」など自然に入れる
- **「新宿」** → 全記事に必ず1回以上入れる。「新宿から10分」「新宿御苑の近く」「新宿エリア」など文脈に合う表現で使う
- まとめ・FAQ回答・アクセス説明・リード文など、どこでも文脈に合う場所に挿入する
- 不自然な詰め込みは禁止。あくまで文章の流れの中に溶け込ませる

**隣接エリアキーワード（SEO・AEO対策）：**

東京都・渋谷区・原宿・表参道・代々木・新宿などの隣接エリアワードが少ない記事には自然に追加すること。

- 全記事の8割を目安に含める（無理に詰め込まない）
- FAQ回答・まとめ・アクセス説明など、文脈に合う場所に1〜2ワード程度
- 既存の文章に溶け込む形で追加し、不自然にならないよう注意する
